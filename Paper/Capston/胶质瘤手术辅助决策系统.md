# 背景

胶质瘤手术的关键矛盾在于：既要最大化安全切除肿瘤，又要尽量保护运动、语言、认知等关键脑功能区。传统术前规划依赖医生对二维影像的经验判断，信息分散、空间关系不直观、个体化评估不足

在真实临床流程中
- 放射科需要精准勾画脑功能区
- 神经外科需要制定手术入路与切除边界
- 放疗科需要确定靶区范围
- 患者和家属也需要理解手术风险

## 核心痛点

当前胶质瘤术前评估存在三个关键问题。

1. 肿瘤边界和周围浸润区域难以仅凭常规影像精确判断，尤其是侵袭性细胞可能已经超出增强区域
2. 肿瘤会破坏正常脑组织结构，影响 FreeSurfer 等自动化脑结构分析流程，导致皮层指标、脑区标签和病理模拟输入不稳定
3. 医生、患者看到的往往是二维切片或分散报告，难以直观理解肿瘤、脑功能区、纤维束和组织形变之间的三维空间关系


# 研究主题

本研究围绕“胶质瘤个体化术前辅助决策”展开
- 深度学习图像分割
- 生成式医学影像修复
- 肿瘤生长数学建模
- 纤维束追踪
- XR 三维交互

项目希望形成从“影像输入”到“手术规划输出”的闭环：输入患者 MRI 多模态影像，输出肿瘤分割、修复后影像、细胞密度分布、组织形变场、关键纤维束和辅助诊断报告

# 研究挑战

本项目面临的核心挑战是跨学科技术链路长、数据形态复杂、临床解释要求高
- 在图像层面，需要稳定处理 t1n、t1c、t2w、t2f 等多模态 MRI，并得到可靠的肿瘤子区域分割结果
- 在结构层面，需要对被肿瘤破坏的脑组织进行合理修复，使后续脑区配准、皮层指标计算和病理模拟具备可用输入
- 在机制层面，需要用反应-扩散、组织弹性和多细胞类型模型描述肿瘤增殖、浸润、坏死、缺氧与占位效应
- 在可视化层面，需要实现多模态数据信息叠加显示，将纤维束与肿瘤三维立体结构关系更好的展示


# 研究问题

本项目重点回答四个问题

1. 如何从多模态 MRI 中自动、稳定地分割胶质瘤及其关键子区域？
2. 如何在保留患者个体结构特征的前提下，对肿瘤破坏区域进行医学影像修复？
3. 如何结合脑区标签、病理生长模型和 MRS 验证，推断肿瘤细胞密度分布及其对周围组织的压力形变？
4. 如何将算法输出转化为 XR 工作站和辅助诊断报告，真正服务于术前规划、风险沟通和临床决策？

## 工作点 1：肿瘤分割

本模块使用 U-Net 系列网络完成脑胶质瘤多模态 MRI 分割，为后续影像修复和病理模拟提供基础输入

Input

Output
- tumor seg (1 2 4)
Network

- [ ] Swin UNETR

## 工作点 2：肿瘤修复

使用 Diffusion Model 对脑胶质瘤破坏区域进行影像修复，目标是重建接近患者正常结构的 T1 影像，为 FreeSurfer SBM/VBM 管线和后续病理模拟提供稳定输入

Input
- t1n
- tumor seg
Output
- t1n_inpainting
Network
- [x] pix2pix -baseline
- [x] fastwdm


## 工作点 3：肿瘤生长病理模拟

本模块使用 reaction-advection-diffusion PDE、linear-screening elasticity 和 multi-species 模型，模拟胶质瘤细胞在脑组织中的繁殖、扩散、浸润、坏死、缺氧与占位效应。
模型将肿瘤相关成分划分为侵袭细胞、浸润细胞、坏死细胞和氧气分布，并结合 atlas labels 与 patient labels 推断患者个体化病理状态

Input
- altas labels
- patient labels
Output
- 肿瘤细胞密度分布图
- 肿瘤对周围组织的挤压形变场
- 肿瘤患者修复后的影像
Valid
- 多体素MRS

## 工作点 4：XR工作站+辅助诊断报告

本模块将肿瘤分割、脑功能区、纤维束、细胞密度、压力形变和皮层指标整合到 XR 工作站与结构化报告中，让复杂医学信息以三维方式呈现。
系统基于脑功能区划分结果，结合 CSD 与确定性纤维追踪，重建 CST、CC 等关键 fiber bundle，并与肿瘤空间位置进行叠加展示

影像
- 纤维束
- 关键纤维束
- 肿瘤分割影像
- 脑功能区       
- 肿瘤细胞密度分布图
- 肿瘤周围组织的压力形变场
- 肿瘤修复后的影像

数值
- 肿瘤大小及范围
- 健康侧与肿瘤侧皮层指标对比（皮层厚度，表面积，体积）
